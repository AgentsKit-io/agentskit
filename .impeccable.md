## Design Context

### Users
JavaScript and TypeScript developers evaluating ready-to-use AI agents. They need to discover a relevant agent, understand its contract and practical usage, compare alternatives, and install it quickly without losing ownership of the code.

### Brand Personality
Technical, calm, and trustworthy. The Registry should feel like a focused developer tool: precise enough for production decisions, approachable enough for quick exploration, and consistent with the wider AgentsKit ecosystem.

### Aesthetic Direction
A restrained, information-first interface in the existing AgentsKit light and dark themes. Prefer clear typography, compact navigation, meaningful category symbols, and purposeful code surfaces. Avoid decorative card walls, visual effects that compete with content, and repetitive icon-plus-description grids.

### Design Principles
1. Put discovery first: search, category, and pagination state must be obvious, fast, and shareable.
2. Show evidence before promotion: agent pages lead with contracts, examples, source, packages, and validation status.
3. Use progressive disclosure: summarize first, then expose implementation detail without overwhelming the page.
4. Preserve developer flow: every important page should connect naturally to related agents, guides, raw manifests, and installation commands.
5. Treat accessibility, responsive behavior, and crawlability as release requirements.
